# Contributing to TSN Test Case Generator

Thank you for your interest in contributing! Please follow these guidelines to help us maintain a high-quality project.

## How to Contribute

- Fork the repository and create your branch from `main` or the latest feature branch.
- Write clear, concise commit messages.
- Ensure your code follows PEP 8 style guidelines.
- Add or update docstrings and comments as needed.
- Add tests for new features or bug fixes.
- Run all tests (`pytest`) and linters (`ruff`, `mypy`) before submitting a PR.
- Submit a pull request with a clear description of your changes and reference any related issues.

## Code Review Process

- All PRs require at least one review before merging.
- Address all review comments and suggestions.
- Squash commits if requested.

## Reporting Issues

- Use GitHub Issues for bugs, feature requests, or questions.
- Include steps to reproduce, expected behavior, and relevant logs or screenshots.

## Project Structure

- `tsn_case_gen_`: Main source code
- `tests/`: Unit and integration tests
- `docs/`: Documentation and architecture
